# 사용 기술 정리

## 프레임워크 / 런타임

### Next.js 16 (App Router)
- **역할**: 리액트 기반 풀스택 프레임워크
- **사용 이유**: Vercel 공식 플랫폼 최적화, App Router로 서버/클라이언트 컴포넌트 분리, 정적 사이트(SSG) 생성으로 빠른 로딩
- **주요 사용처**: 라우팅, 레이아웃, 메타데이터 설정, dynamic import

### React 19
- **역할**: UI 컴포넌트 라이브러리
- **사용 이유**: Next.js 기반, 컴포넌트 단위 개발

### TypeScript 5
- **역할**: 정적 타입 검사
- **사용 이유**: 컴포넌트 Props 타입 보장, 런타임 오류 사전 차단

---

## 스타일링

### Tailwind CSS v4
- **역할**: 유틸리티 기반 CSS 프레임워크
- **사용 이유**: 빠른 스타일 개발, 다크 테마 구성, 반응형 레이아웃
- **v4 특징**: `@import "tailwindcss"` 방식, `@theme` 디렉티브로 커스텀 변수 정의
- **주요 사용**: 색상(slate, indigo, emerald, violet 등), 레이아웃(grid, flex), 반응형(sm:, md:, lg:)

---

## 애니메이션

### Framer Motion v12
- **역할**: 리액트 애니메이션 라이브러리
- **사용 이유**: 선언적 API로 복잡한 애니메이션을 간결하게 표현, React 19와 호환
- **사용 기능**:

| 기능 | 사용처 |
|------|--------|
| `motion.div` / `motion.button` | 모든 애니메이션 요소 |
| `whileInView` + `viewport={{ once: true }}` | 스크롤 진입 시 트리거 애니메이션 |
| `variants` + staggerChildren | 카드 그리드 순차 등장 |
| `layoutId` | 네비게이션 활성 인디케이터 슬라이딩 |
| `useScroll` | 스크롤 진행률 추적 |
| `initial` / `animate` / `transition` | 기본 진입 애니메이션 |
| `whileHover` / `whileTap` | 인터랙티브 hover/클릭 효과 |

## 스크롤

### Lenis v1.3
- **역할**: 부드러운 관성 스크롤 라이브러리
- **사용 이유**: 브라우저 기본 스크롤보다 훨씬 부드러운 UX 제공
- **설정**: `duration: 1.2`, 커스텀 easing 함수 (`1.001 - 2^(-10t)`)
- **구현**: `LenisProvider` 클라이언트 컴포넌트로 래핑, `requestAnimationFrame` 루프

## 파티클 / 배경 효과

### tsParticles (@tsparticles/react + @tsparticles/slim) v3
- **역할**: Hero 섹션 인터랙티브 파티클 배경
- **사용 이유**: AI 네트워크 느낌의 연결 선 파티클 효과
- **설정**:
  - 파티클 색상: 인디고 (`#818cf8`)
  - 연결 선: 거리 160px, 불투명도 12%
  - 마우스 호버 시 주변 파티클과 연결 강조 (grab 모드)
  - 이동 속도: 0.6 (느리고 우아한 움직임)
- **주의**: SSR 비호환 → `dynamic(() => import(...), { ssr: false })`로 클라이언트 전용 렌더링

## 배포

### Vercel
- **역할**: Next.js 앱 호스팅 플랫폼
- **사용 이유**: Next.js 공식 개발사 운영, GitHub 연동 자동 배포, 무료 플랜에서 충분히 사용 가능
